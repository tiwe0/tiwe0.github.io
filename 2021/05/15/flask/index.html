1.3 路由及视图函数

路由规则表: app.urlmap

  • 路由规则表本质上是哈希映射表 key 部分为路由路径，value 部分为对应的视图函数
  • 将视图函数加入路由规则表的方法有两种：

    • 先定义视图函数，再使用类方法将视图函数注册到路由规则表

    • 在定义视图函数时，使用 app 下的装饰器函数直接注册

1.4 蓝图

在搭建小的 web 应用时，直接添加路由不失为一个简便的方法。 但当 web 应用逐渐增大时，这会导致管理混乱，效率低下，出错率增大。 为了系统的管理路由及请求错误处理，Flask 使用了蓝图。 可以将蓝图视为模块式管理视图的一种方法。 蓝图的创建和 app 类似

蓝图的使用和 app 类似，使用装饰器将其注册到路由表中

蓝图通过前后缀等方式来避免路由路径的冲突。

1.5 框架逻辑

在较大的 web 框架下，由工厂函数流水线般半动态生成 Flask 应用——app

  1. 实例化 Flask 应用
  2. 配置 Flask 应用
  3. 注册插件（以 app 作为参数。前端框架，邮件系统，数据库等）
  4. 注册蓝图（给 app 注册蓝图。路由，视图函数等）
  5. 返回 app

1.6 一个请求的旅途

  1. 客户端将请求发送到 web 服务器
  2. 请求经过 WSGI 协议，封装成环境，并传递给 Flask 应用 app
  3. app 创建一个新的线程来处理请求
  4. 激活请求上下文。
  5. app 将封装环境，并从环境提取出 request 信息并封装成 request 对象
  6. 根据路由规则表将 request 传递给对应的视图函数
  7. 视图函数相应请求，并返回响应
  8. app 将响应封装成 response，再通过 WSGI 协议发送给 web 服务器
  9. 关闭请求上下文，并关闭线程。
  10. web 服务器作出响应
