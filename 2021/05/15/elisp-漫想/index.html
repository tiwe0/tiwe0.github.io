1 Emacs Lisp API 的设计思路

1.1 发散笔记

1.1.1 buffer

  • buffer是一个Lisp对象，其中包含可以编辑的文字，且自带一个buffer-local变量域
  • 通常总有一个正在使用debuffer，使用current-buffer获取当前的buffrt
  • buffer具有名字，使用buffer-name获取buffer的名字，使用set-buffer可以给当前buffer修改名字，使用rename-buffer更改buffer的名字
  • 可以使用save-current-buffer, with-current-buffer, with-temp-buffer进行关于buffer的上下文管理
  • buffer的名字终究不是buffer本身，使用get-buffer来获取buffer本身
  • emacs中可能有很多的buffer，名字也有可能冲突，使用generate-new-buffer-name来得到一个可用的 buffer名字
  • 通常buffer是修改文件的临时港湾，使用buffer-file-name可以获取当前buffer对应的文件名，同时它还有个变量形式
  • 不过以上的名字并不是文件的真名，而是有可能是符号链接，管道啥的，因此其真名为buffer-file-truename
  • 使用名字毕竟不是计算机的习惯，可以查看其名字对应的数字buffer-file-number
  • 既然可以查找buffer对应的文件，那也可以查找文件被哪个buffer对应，使用get-file-buffer
  • 相信你经常记不住文件的名词，没关系，使用find-buffer-visiting进行模糊搜索
  • 既然存在，那就可以修改，使用set-visited-file-name修改当前buffer对应文件的名称
  • variable: list-buffers-directory -—> TODO
  • 使用buffer-modified-p来判断当前buffer有没有修改过
  • 使用set-buffer-modified-p来直接修改当前buffer有没有修改过，釜底抽薪
  • 不过前者有一个副作用，会改变mode line，使用restore-buffer-modified-p可以规避
  • 使用not-modified快速标记当前buffer为被修改，比你藏手机还要快
  • 仅仅知道buffer有没有修改过可不够，使用buffer-modified-tick来查看修改的次数（可能）
  • 使用buffer-chars-modified-tick查看字符修改的次数
  • 还有一个谜之宏with-silent-modifications
  • 可是文件也不是排他的，也有可能有其他软件访问修改，使用verify-visited-file-modtime来查看emacs记录的时间和操作系统记录的时间是否相同
  • 使用clear-visited-file-modtime来清除上次修改的时间
  • 使用visited-file-modtime来查看最近一次修改的时间
  • 当然，用户就是上帝，你也可以使用set-visited-dile-modtime来篡改这个时间
  • 有时候你会遇到这种情况一个文件比当前buffer内容还要新，那究竟是使用现在buffer内的内容呢，还是使用新文件的内容呢（如果你是vim用户，这种情况不可谓不常见），这时，使用ask-user-about-supersession-threat来拷问用户
  • 在emacs里，有不少的buffer是只读的，这是必要的。查看buffer-read-only变量来判断当前buffer是不是只读。
  • .。。。inhibit-read-only
  • 你可以使用read-only-mode来手动修改是否只读
  • 当buffer是只读时，barf-if-buffer-read-only会抛出一个错误，到底使用来干啥的。。。
  • 当你可以处理一个buffer时，你就可以处理一堆buffer了
  • 使用buffer-list可以查看所有的buffer的一个列表，没错，是列表，yyds
  • 有时候你只想针对其他的buffer，使用other-buffer即可（不过实际上这个只返回buffer列表里第一个其他buffer
  • last-buffer人如其名，返回上一个buffer
  • bury-buffer把buffer放到buffer列表的最后。
  • unbury-buffer直接切换到buffer列表最后一个buffer，
  • 为了监控buffer列表的buffer们，可以查看buffer-list-update-hook这个变量，监视这个变量的除了你，同时还有get-buffer-create，rename-buffer，kill-buffer，bury-buffer和select-window
  • 说了这么多，怎么创建buffer呢，使用generate-new-buffer来创建新的buffer，但是并不改变当前的buffer
  • 使用get-buffer-create可以获得一个指定的buffer，不过，如果不存在的话，那会得到一个新的。这里可以规避命名重复
  • 创造和毁灭是一个硬币的两面，使用kill-buffer来杀死一个buffer（你的良心不会痛吗），释放所有的内存归还给操作系统，任何和这个buffer有联系的进程都会被挂起
  • 不过怎么处理后续的进程呢？在调用kill-buffer后，kill-buffer会调用kill-buffer-query-functions列表内的函数
  • 此外有个kill-buffer-hook跟踪buffer的销毁
  • 有的buffer删除后会保存或者不保存当前内容，当他为非nil时，这时就不会询问用户，而是直接保存
  • 你可以使用buffer-live-p来查询一个buffer是不是还活着
  • 有些buffer是间接buffer，所谓间接，就是指共享内容的buffer
  • 使用make-indirect-buffer来创建间接buffer
  • 你也可以使用clone-indirect-buffer来克隆当前buffer的一个间接buffer
  • 使用buffer-base-buffer来获取buffer的base-buffer，就是间接buffer里最底层的那个。如果这个buffer不是间接buffer，那就会返回nil
  • 你可以使用buffer-swap-text来交换两个buffer的内容
  • 实际上，emacs使用了一个称之为gap的东西来加速buffer中的插入和删除，使用gap-position来查看当前gap的位置，使用gap-size来查看当前buffer内gap的大小

1.2 API 设计思路

  • save-current-buffer
  • with-current-buffer 上下文管理宏
    1. 临时将当前 current-buffer 设置为指定 buffer
    2. 执行代码主题
    3. 退出后恢复原 current-buffer
  • with-temp-buffer 临时环境宏
    1. 创建一个临时 buffer
    2. 将这个临时 buffer 设为 current-buffer
    3. 执行代码主体
    4. 退出环境后销毁这个临时 buffer
    5. 恢复原来的 current-buffer
  • kill-buffer 关闭 buffer
  • rename-buffer 重命名 buffer
  • get-buffer 返回 buffer
  • generate-new-buffer-name 生成一个独有的 buffer 名字，但不生成新 buffer
  • buffer-file-name : function & variable
  • buffer-file-trueame :variable
  • buffer-file-number :variable
  • get-file-buffer :function 返回一个访问某个文件的 buffer
  • find-buffer-visiting :function 返回一个访问某个文件的 buffer，但使用的是模糊搜索
  • set-visited-file-name :function 设置当前 buffer 访问的文件
  • list-buffer-directory :variable
  • buffer-modified-p :function 若 buffer 内容改变了，则返回 t；这里的改变适合上次读取或保存进行比较
  • set-buffer-modified-p :function 手动改变 buffer 是否改变的标记
  • restore-buffer-modified-p :function 和上面的相似，但是不强制 redisplay of mode lines
  • not-modified :command 标记当前 buffer 是未发生改变的。
  • buffer-modified-tick :function buffer 变动记录
  • buffer-chars-modified-tick :function 变动记录
  • with-silent-modifications :macro
  • verify-visited-file-modtime :function
  • clear-visit-file-modtime :function
  • visited-file-modtime :function
  • set-visited-file-modtime :function
  • ask-user-about-supersession-thread :function
  • buffer-read-only :variable
  • inhibit-read-only :variable
  • read-only-mode :command
  • barf-if-buffer-read-only :function
  • buffer-list :function
  • other-buffer :function
  • last-buffer :function
  • bury-buffer :command
  • inbury-buffer :command
  • buffer-list-update-hook :variable
  • get-buffer-create :function
  • generate-new-buffer :function
  • kill-buffer :command
  • kill-buffer-query-functions :variable
  • kill-buffer-hook :vairable
  • buffer-offer-save :uer option
  • buffer-save-without-query :variable
  • buffer-live-p :function
  • make-indirect-buffer :command
  • clone-indirect-buffer :command
  • buffer-base-buffer :function
  • buffer-swap-text-buffer :function
  • gap-position :function
  • gap-size :function
